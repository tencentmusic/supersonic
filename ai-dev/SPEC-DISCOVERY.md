# SPEC-DISCOVERY.md — SPEC 四阶段对话生成流程

## 使用场景
在创建或修改 intent.md 之前，必须先走完此对话流程。

---

## Phase 1：业务边界探索（AI 追问 10 轮）

### 目标
AI 扮演「好奇的新人开发者」，通过追问暴露需求中的隐含假设和业务边界。

### SuperSonic 追问清单（按模块）
- **语义层（headless）**：NL2SQL 解析策略？LLM fallback 机制？语义模板结构？
- **对话层（chat）**：Agent 路由规则？多轮对话上下文管理？
- **权限层（auth）**：RBAC 粒度？租户隔离级别？
- **数据层（common）**：MyBatis-Plus 分页策略？缓存失效机制？

### 输出
- 业务边界澄清列表
- 隐含假设清单

---

## Phase 2：破坏者模式（AI 挖 Anti-Goals）

### 目标
AI 扮演「破坏者」，尝试找出系统中可能被滥用或误用的路径。

### SuperSonic 重点破坏方向
- 跨租户数据泄露：能否通过 API 参数注入访问其他租户数据？
- NL2SQL 注入：自然语言输入能否触发危险 SQL？
- Agent 权限逃逸：Agent 能否执行超出其数据集范围的查询？
- 模板篡改：低权限用户能否修改语义模板？

### 输出
- Anti-Goals 列表（每条标注严重等级）

---

## Phase 3：合规审计模式

### 目标
AI 扮演「合规审计员」，审查数据敏感级别和合规要求。

### SuperSonic 合规重点
- 用户查询日志是否含敏感数据？
- LLM API 调用是否传输了敏感业务数据？
- 导出功能是否有数据量限制？
- 多租户数据物理隔离 vs 逻辑隔离？

### 输出
- 数据敏感级别标注
- 合规风险清单

---

## Phase 4：SPEC 文档生成

### 输入
Phase 1-3 的所有输出

### 输出
- ai-spec/domain/intent.md（AI 起草，人工审定）
- SCRATCHPAD 初始条目（含 SPEC 探索推理链）
