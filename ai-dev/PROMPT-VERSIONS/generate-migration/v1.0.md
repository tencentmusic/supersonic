## 意图
生成符合 SuperSonic 项目规范的 Flyway 迁移脚本。
必须同时生成 MySQL 和 PostgreSQL 两个版本，并遵守各方言的语法约束。

## 上下文加载顺序
1. @CLAUDE.md（Flyway 规则 + MySQL vs PostgreSQL 差异）
2. @ai-spec/domain/entities.md（字段定义）
3. 现有最新迁移文件（mysql/ 和 postgresql/ 各取最新一个，确认下一版本号）
4. @ai-spec/domain/state-machines.yaml（如涉及状态字段）

## 任务描述
生成：
- `launchers/standalone/src/main/resources/db/migration/mysql/V{N}__{description}.sql`
- `launchers/standalone/src/main/resources/db/migration/postgresql/V{N}__{description}.sql`

## 强制约束（SuperSonic 专项）

### 版本号规则
- 新迁移从 V21 起，检查现有文件后选择下一个可用版本号
- 两个方言文件必须使用相同版本号

### MySQL 约束
- 禁止 `ADD COLUMN IF NOT EXISTS`（MySQL 不支持，仅 MariaDB/PostgreSQL 支持）
- 如需幂等性 ADD COLUMN，使用存储过程检查 `information_schema.columns`，或依赖 Flyway 单次执行保证
- 使用 `AUTO_INCREMENT` 作为自增主键
- 使用反引号作为标识符引用（可选）
- 使用 `TINYINT(1)` 表示布尔值
- `ENGINE=InnoDB DEFAULT CHARSET=utf8mb4`

### PostgreSQL 约束
- 支持 `ADD COLUMN IF NOT EXISTS`（可用于幂等性）
- 使用 `SERIAL` 或 `BIGSERIAL` 作为自增主键（或 `GENERATED ALWAYS AS IDENTITY`）
- 使用双引号作为标识符引用（仅在必要时）
- 使用 `BOOLEAN` 表示布尔值
- 不需要 ENGINE 子句

### 通用规则
- `CREATE TABLE` 必须使用 `IF NOT EXISTS`
- 租户隔离表必须包含 `tenant_id VARCHAR(64)` 字段
- 字段命名使用 snake_case

## 验收条件
- [ ] MySQL 和 PostgreSQL 两个文件均已生成
- [ ] 两个文件使用相同的版本号（V21 或更高）
- [ ] MySQL 版本中不包含 `ADD COLUMN IF NOT EXISTS`
- [ ] PostgreSQL 版本中不包含 `AUTO_INCREMENT` 或 `ENGINE=InnoDB`
- [ ] `CREATE TABLE` 使用 `IF NOT EXISTS`
- [ ] 租户隔离表包含 `tenant_id` 字段
- [ ] 字段定义与 entities.md 一致

## 输出格式
1. 《迁移决策说明》（版本号选择理由 + 方言差异处理）
2. MySQL 迁移文件（完整 SQL）
3. PostgreSQL 迁移文件（完整 SQL）
