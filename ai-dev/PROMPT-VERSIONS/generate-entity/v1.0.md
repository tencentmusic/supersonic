## 意图
生成符合 SuperSonic 项目规范的数据层代码（DO + Mapper + Flyway 迁移脚本）。

## 上下文加载顺序
1. @ai-spec/domain/entities.md
2. @CLAUDE.md（MyBatis-Plus 约定）
3. 已有的 DO 类示例

## 任务描述
生成：DO 类（@TableName/@TableId + Lombok）+ Mapper（BaseMapper）+ Flyway 迁移脚本（MySQL + PostgreSQL）

## 强制约束（SuperSonic 专项）
- DO 类：@TableName + @TableId + Lombok @Data
- Mapper：extends BaseMapper<DO>
- 多租户：tenant_id 字段（VARCHAR(64)），排除表除外
- Flyway：V21+ 版本号，同时提供 MySQL 和 PostgreSQL 版本
- MySQL 迁移：不使用 `ADD COLUMN IF NOT EXISTS`（MySQL 不支持）
- 使用 `IF NOT EXISTS` 保证幂等（CREATE TABLE）

## 验收条件
- [ ] DO 类包含 @TableName 和 @TableId 注解
- [ ] DO 类包含 tenant_id 字段
- [ ] Mapper extends BaseMapper<DO类>
- [ ] Flyway 迁移脚本包含 MySQL 和 PostgreSQL 两个版本
- [ ] MySQL 版本不包含 `ADD COLUMN IF NOT EXISTS`
- [ ] 字段命名与 entities.md 一致
- [ ] state-machines.yaml 中的状态在 DO 中有对应字段

## 输出格式
1. DO 类文件
2. Mapper 接口文件
3. Flyway 迁移脚本（MySQL + PostgreSQL）
