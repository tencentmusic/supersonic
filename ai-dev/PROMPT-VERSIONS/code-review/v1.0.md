## 检查维度

### 1. 架构合规性（严重级）
- @Autowired 字段注入 → 应为 @RequiredArgsConstructor
- Controller 包含业务逻辑 → 应下沉到 Service
- Class.forName 跨模块反射 → 应用 ApplicationEvent
- DO 类缺少 tenantId → 多租户隔离缺失

### 2. 业务合规性（严重级）
- Anti-Goals 违反
- 状态机非法跳跃未防护
- 缺少对应测试覆盖

### 3. 代码质量（一般级）
- 方法超过 20 行
- if 嵌套超过 3 层
- 重复代码模式 > 3 次
- 前端时间列直接渲染原始 `createdAt` / `updatedAt` / ISO 字符串，未使用 `dayjs(value).format('YYYY-MM-DD HH:mm:ss')`
- 前端 service 硬编码 `/api/semantic/` 等路径，未使用 `process.env.API_BASE_URL` 或 `AUTH_API_BASE_URL`
- TypeScript API 类型使用 `any`，未按接口定义命名类型

### 4. 建议级
- 命名优化
- 注释补充
- @AiGenerated 标注建议

## 输出格式
按「严重 / 一般 / 建议」三级分类输出问题列表，每个问题标注违反的具体规范来源。

前端改动追加要求：
- 表格里凡是时间字段（如 `createdAt`、`updatedAt`、`startedAt`、`completedAt`），若没有显式 `render` 格式化，必须报问题
- 不允许把“页面能显示”视为通过；违反仓库前端约定也必须报问题
