## 强制约束（SuperSonic 专项）

1. **时间戳格式**：表格列中所有时间戳必须使用 `dayjs(value).format('YYYY-MM-DD HH:mm:ss')` 渲染，禁止直接渲染原始 ISO 8601 字符串
2. **API base URL**：所有 HTTP 请求路径必须使用 `process.env.API_BASE_URL` 或 `AUTH_API_BASE_URL`，禁止硬编码 `/api/semantic/` 等路径
3. **复制现有模式**：页面结构和组件模式必须与指定的 pattern 参考页面完全一致，禁止引入新的布局、样式或 Ant Design 组件
4. **TypeScript 类型**：所有 API 响应对象和请求对象必须有完整的 TypeScript interface 定义，禁止使用 `any` 类型
5. **request 工具**：所有 HTTP 调用必须使用项目的 `request` 工具函数，禁止直接使用原生 `fetch()` 或 `axios`
6. **render 函数**：表格列中凡是需要变换展示的字段（日期、枚举、状态、布尔值）必须提供显式的 `render` 函数

## 验收条件
- [ ] 生成的组件文件结构与 pattern 参考页面一致
- [ ] 所有时间戳列使用 `dayjs(value).format('YYYY-MM-DD HH:mm:ss')`
- [ ] service.ts 中所有请求使用 `process.env.API_BASE_URL` 或 `AUTH_API_BASE_URL`
- [ ] 所有 API 响应类型已定义为命名 TypeScript interface
- [ ] 无 `any` 类型出现（TypeScript 类型完整）
- [ ] HTTP 调用使用项目 `request` 工具，无原生 `fetch()` 调用
- [ ] 需要变换展示的表格列有显式 `render` 函数
- [ ] 路由配置更新已包含在输出中

## 输出格式
1. 《生成决策说明》（pattern 参考选择理由 + 与 openapi 契约的对应关系）
2. 页面组件文件（`pages/{FeatureName}/index.tsx`）
3. 服务 API 文件（`pages/{FeatureName}/service.ts`）
4. 路由配置更新片段（`config/routes.ts` 对应条目）
