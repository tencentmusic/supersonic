## 八、架构决策说明

1. **Quartz 内嵌部署**：通过 `spring-boot-starter-quartz` + JDBC JobStore，无需额外基础设施。集群模式通过数据库行锁实现任务防重复。
2. **Export Worker 独立线程池**：`exportExecutor`（3线程）与 Web 线程池、Quartz 线程池完全隔离，导出任务不影响正常请求处理。
3. **Quartz Job 轻量分派**：`ReportScheduleJob` 和 `DataSyncJob` 仅负责设置 TenantContext 和调用 Service，不包含业务逻辑。
4. **QuartzJobManager 组合模式**：将 ReportScheduleServiceImpl 和 DataSyncServiceImpl 中重复的 Quartz Scheduler 操作（创建/暂停/恢复/删除/触发/重调度 Job）提取到 `QuartzJobManager` 工具类，通过组合而非继承消除代码重复，保持 ServiceImpl 继承链不变。
5. **线程池隔离策略**：报表调度（REPORT 组，5 线程）与数据同步（DATA_SYNC 组，@DisallowConcurrentExecution）使用不同 Quartz Job Group。
5. **多租户兼容**：所有新表均包含 `tenant_id` 字段，Quartz Job 在执行时通过 JobDataMap 传递 tenantId 并设置 TenantContext。

## 十、SQL 级查询超时实现（2026-02-05）

本次调整完成了查询资源隔离中的 SQL 级超时功能，防止长时间运行的查询占用数据库资源。

### 10.1 实现方案

使用 JDBC 标准的 `Statement.setQueryTimeout()` 机制，通过 Spring `JdbcTemplate.setQueryTimeout()` 统一注入。此方案：

- **跨数据库兼容**：适用于所有 JDBC 驱动（MySQL、PostgreSQL、ClickHouse、H2 等）
- **驱动级执行**：超时由 JDBC 驱动在网络层强制执行，无需数据库特定 SQL 语法
- **可配置**：通过 `application.yaml` 配置项 `s2.source.query-timeout` 调整

### 10.2 修改文件

| 文件 | 变更说明 |
|------|----------|
| `headless/core/src/main/java/com/tencent/supersonic/headless/core/utils/SqlUtils.java` | 新增 `@Value("${s2.source.query-timeout:30}")` 配置项；`jdbcTemplate()` 方法中调用 `jdbcTemplate.setQueryTimeout(queryTimeout)`；`SqlUtilsBuilder` 新增 `withQueryTimeout()` 方法和字段传递 |

### 10.3 配置说明

```yaml
# application.yaml
s2:
  source:
    query-timeout: 30  # 查询超时（秒），0 表示无超时，默认 30 秒
```
