## 1. 全链路联调（优先级最高）

### 1.1 联调范围

单模板 × MySQL × 结构化查询 × 飞书推送，覆盖以下完整链路（SqlTemplateConfig 路径一期前端无入口，不纳入联调范围）：

```
创建模板 → 部署 → 手动执行 → 查看结果
                → 创建调度 → 等待触发 → 查看执行记录
                → 导出（同步 + 异步）→ 下载文件
                → 飞书推送 → 验证卡片内容
                → 模拟失败 → 验证重试 → 验证自动禁用
```

## 2. 指标与告警落地（优先级第二）

### 2.1 部署方案

两种部署形态并存：

**形态 A：docker-compose 自包含**

在 `docker/` 目录下新增 `docker-compose-monitoring.yml`：

```yaml
services:
  prometheus:
    image: prom/prometheus:v2.51.0
    volumes:
      - ./prometheus/prometheus.yml:/etc/prometheus/prometheus.yml
      - ./prometheus/rules/:/etc/prometheus/rules/
    ports:
      - "9090:9090"

  grafana:
    image: grafana/grafana:10.4.0
    volumes:
      - ./grafana/provisioning/:/etc/grafana/provisioning/
      - ./grafana/dashboards/:/var/lib/grafana/dashboards/
    ports:
      - "3000:3000"
    environment:
      - GF_SECURITY_ADMIN_PASSWORD=admin

  alertmanager:
    image: prom/alertmanager:v0.27.0
    volumes:
      - ./prometheus/alertmanager.yml:/etc/alertmanager/alertmanager.yml
    ports:
      - "9093:9093"
```

**形态 B：对接公司级平台**

应用仅暴露 `/actuator/prometheus` 端点，由公司级 Prometheus 主动拉取。需要：
- 确保 `/actuator/prometheus` 端点在生产环境仅内网可访问（AG-13 + 安全）
- 提供 Grafana 面板 JSON 文件，导入公司级 Grafana

### 2.2 告警规则

文件：`docker/prometheus/rules/report-slo-alert-rules.yml`

```yaml
groups:
  - name: report-slo
    rules:
      - alert: ReportExecutionSuccessRateLow
        expr: |
          (
            sum(rate(supersonic_report_execution_total{status="SUCCESS"}[10m]))
            /
            sum(rate(supersonic_report_execution_total[10m]))
          ) < 0.95
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "报表执行成功率低于 95%"

      - alert: ReportExecutionP95High
        expr: |
          histogram_quantile(0.95,
            rate(supersonic_report_execution_duration_seconds_bucket[10m])
          ) > 10
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "报表执行 P95 时延超过 10 秒"

      - alert: ExportTaskPendingHigh
        expr: supersonic_report_export_pending > 10
        for: 10m
        labels:
          severity: warning
        annotations:
          summary: "导出任务积压超过 10 个（持续 10 分钟）"

      - alert: DeliveryFailRateHigh
        expr: |
          (
            sum(rate(supersonic_report_delivery_total{status="FAILED"}[30m]))
            /
            sum(rate(supersonic_report_delivery_total[30m]))
          ) > 0.05
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "推送失败率超过 5%"

      - alert: DeliveryConfigAutoDisabled
        expr: supersonic_report_delivery_config_disabled > 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "存在被自动禁用的推送配置"
```

注意：所有规则都带 `for` 持续条件（B-09），避免瞬时抖动触发告警风暴。

### 2.4 指标 tag 安全检查（AG-13）

验证 `TemplateReportMetrics` 中所有指标的 tag 只使用低基数字段：

| 允许的 tag | 基数 |
|-----------|:---:|
| `status` | ≤ 5 |
| `trigger_type` | ≤ 4（MANUAL / SCHEDULE / AGENT / API） |
| `channel_type` | ≤ 5（EMAIL / FEISHU / DINGTALK / WECHAT_WORK / WEBHOOK） |
| `export_format` | ≤ 3（EXCEL / CSV / JSON） |

| 禁止的 tag | 原因 |
|-----------|------|
| `template_id` | 模板数量无上限，时序膨胀 |
| `user_id` | 用户数量无上限 |
| `tenant_id` | 租户数量可能很多 |

## 4. Runbook 完善（优先级第四）

### 4.1 文档结构

文件：`docs/runbook/report-operations.md`

每个场景统一格式：

```markdown
### 场景名称

**现象**：运营/运维看到了什么
**影响**：影响范围和紧急程度
**排查步骤**：
1. 具体命令（可复制粘贴）
2. 看什么输出、判断什么
**止损操作**：
1. 具体命令
2. 影响范围说明
3. 回滚步骤
```

### 4.2 一期覆盖的 4 个场景

| 场景 | 现象 | 关键排查命令 |
|------|------|-------------|
| 调度失败 | 执行记录状态 FAILED | 查 `s2_report_execution` → 看 error_message → 检查连接池/超时配置 |
| 导出异常 | 导出任务长时间 PENDING 或 FAILED | 查 `s2_export_task` 状态 → 检查 EXPORT 线程池使用率 → 检查磁盘空间 |
| 推送失败 | 飞书群没收到消息 | 查 `s2_report_delivery_record` → 看 error_message → 检查 Webhook URL 有效性 → 检查是否被自动禁用 |
| 推送被自动禁用 | 告警触发 `DeliveryConfigAutoDisabled` | 查 `s2_report_delivery_config` disabled 原因 → 修复根因 → 手动恢复启用 |

## 5. 压测与阈值校准（优先级第五）

### 5.1 一期范围：功能级基准测试

在 CI 中用 JUnit 验证以下行为正确性：

| 测试项 | 验证内容 | 实现方式 |
|--------|---------|---------|
| 连接池隔离 | EXPORT 池满时 INTERACTIVE 池不受影响 | 并发提交导出任务占满 EXPORT 池，同时发起交互查询 |
| 导出阈值分流 | < 阈值走同步，≥ 阈值走异步 | 构造不同行数的结果集，验证分流行为 |
| 调度并发限制 | 租户并发 > 5 时排队（AG-14） | CountDownLatch 模拟 6 个并发调度任务 |
| Quartz Misfire | 应用重启后错过的任务按策略补跑或跳过 | 注册任务 → 停止 Scheduler → 等待错过 → 重启 → 验证行为 |
| MySQL EXPLAIN 估算偏差 | RowCountEstimator 对 MySQL 的估算偏差在 5 倍以内 | 准备已知行数的表，对比估算值和实际值 |
